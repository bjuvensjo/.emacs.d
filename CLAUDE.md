## What this is

Personal Emacs configuration for Magnus Bjuvensjo. Written in Emacs Lisp (elisp). No build system, tests, or linting pipeline — changes are validated by loading the config in a running Emacs instance.

## How to reload/test changes

```bash
# Reload a specific settings file from within Emacs:
M-x load-file RET settings/setup-package.el RET

# Reload init.el entirely (requires restart or eval-buffer):
M-x eval-buffer  # from init.el buffer

# Check for elisp syntax errors:
M-x check-parens  # from any elisp buffer
```

## Architecture

**Entry point:** `init.el` — bootstraps package.el with MELPA, then loads all modules via `require` or direct load calls.

**Module layout:**

| Directory | Purpose |
|-----------|---------|
| `settings/` | Feature-specific configuration modules (package setup, keybindings, appearance, etc.) |
| `defuns/` | Generic helper functions (buffer, editing, file, lisp, misc) |
| `my-defuns/` | Personal custom functions (run-current-file, window toggling, sorting, zoom) |
| `wiki-defuns/` | Third-party utility functions (e.g., frame-fns.el) |
| `themes/` | Custom color themes (espresso, zenburn, prez, default-black) |
| `snippets/` | Yasnippet templates |
| `custom.el` | Auto-generated by Emacs Custom; do not edit manually |

**Key settings files:**
- `settings/setup-package.el` — package list and MELPA configuration
- `settings/key-bindings.el` — all custom keybindings (uses key-chord for digraph bindings)
- `settings/appearance.el` — themes, fonts (Andale Mono 12), UI prefs
- `settings/sane-defaults.el` — core Emacs behavior overrides
- `settings/mac.el` — macOS-specific adjustments

## Package management

Uses `package.el` with MELPA. Packages are declared in `settings/setup-package.el` and installed via `package-install` on first load. The `elpa/` directory is gitignored.

To add a package: add it to the package list in `settings/setup-package.el`, then `M-x package-refresh-contents` and `M-x package-install`.

## Keybinding conventions

- Uses `key-chord-mode` for two-key digraph bindings (e.g., `jk`, `df`)
- Super (`s-`) and Hyper (`H-`) modifiers used heavily for custom commands
- Swedish keyboard layout influences some bindings (C-ä, M-ä, C-å style chords appear in comments)
- All custom bindings live in `settings/key-bindings.el`
